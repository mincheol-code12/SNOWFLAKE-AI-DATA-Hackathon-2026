# 상품 inflow 전략 재검증 프로젝트

기존 inflow 전략 분석의 **전제**를 가설로 분해해 검증하고, 동등 비교군 기준으로 기대효과를 재산정해 **월 2.47억 → 2.06억(-17%)**으로 보수화한 분석 프로젝트입니다.

---

## 프로젝트 개요

이 프로젝트는 기존 상품 inflow 전략 분석에서 사용한 핵심 전제를 다시 검토하고, 이를 검증 가능한 가설 단위로 분해해 재분석한 작업입니다.

1차 분석에서는 상품 × inflow별 성과 차이를 바탕으로 레버를 도출하고 기대효과를 추정했습니다.  
2차 분석에서는 아래 두 질문을 중심으로 전제를 재검증했습니다.

- 저성과의 원인이 실제로 **inflow 구조**에 있는가?
- 성과가 좋은 inflow로 이동할 경우, 해당 **CVR이 동일하게 재현되는가?**

그 결과, 일부 기대효과가 과대추정되었음을 확인했고, 최종 기대효과를 **월 2.06억** 수준으로 보수적으로 재산정했습니다.

---

## 핵심 결과

- **인터넷**  
  저성과 병목은 콜센터 자체가 아니라 **상담 이후 등록(C→R) 단계**에서 발생했습니다.  
  따라서 바로가입 유도보다 **바로상담 유도**가 더 타당한 전략으로 해석했습니다.

- **모바일**  
  랜딩 저성과의 주요 원인은 **저관여 유입 혼합**으로 판단했습니다.  
  따라서 **방문고객 유도**와 **혜택 공통화**를 결합한 접근이 필요합니다.

- **렌탈**  
  현재 구조가 비교적 안정적이며, 기존 레버를 유지하는 방향이 적절하다고 판단했습니다.

- **최종 기대효과**  
  월 **2.06억**
  - 인터넷: **1.30억**
  - 모바일: **0.54억**
  - 렌탈: **0.22억**

```text
.
├── product_inflow_strategy_analysis_step0_step1.ipynb   # 1차: 원본 데이터 분석 노트
├── product_inflow_hypothesis_true.ipynb                 # 2차: 가설 검증 노트
└── README.md
```

### 1) `product_inflow_strategy_analysis_step0_step1.ipynb`
상품 × inflow 관점에서 전체 성과 구조를 파악하고, 1차 레버 후보를 도출한 분석 노트입니다.  
초기 기대효과는 **월 2.47억**으로 산정했습니다.

### 2) `product_inflow_hypothesis_true.ipynb`
1차 분석에서 사용한 두 전제를 가설로 분해해 검증한 노트입니다.

- 전제 1: 저성과 원인은 inflow 구조에 있다.
- 전제 2: inflow를 옮기면 높은 CVR이 재현된다.

검증 결과를 바탕으로 레버를 재설계했고, 기대효과를 **월 2.06억**으로 보수화했습니다.

## 해석 시 유의사항

- 본 분석은 **집계 데이터 기반 분석**입니다.
- 따라서 개인 수준 행동 경로나 인과를 직접 식별하는 데에는 한계가 있습니다.
- 기대효과는 **동등 비교군 기준**으로 재산정한 값이며, 실제 실행 성과와는 차이가 있을 수 있습니다.
- 결과 해석은 **전략 우선순위 설정**과 **추가 검증 방향 제시**에 초점을 둡니다.
